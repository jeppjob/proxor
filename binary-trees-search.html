Binary Trees: Binary Search Trees

Binary search trees store collections that are ordered. Insertion, seaching and deletion are all fast and don’t take a lot of space. In this example all data that is less than or equal to a node inserts somewhere in the left sub-tree of the node.

Everything else goes to the right.

A visual example (above):

The sequence of digits, {4,1,2,3,5,2,6,7,2,7}. 

{“FOUR,SCORE,AND,SEVEN,YEARS,AGO”} If node datum <= insert left o/w right
